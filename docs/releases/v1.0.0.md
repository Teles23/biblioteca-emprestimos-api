# v1.0.0 - Backend completo da API de Biblioteca

Data: 2026-03-04  
Tag: `v1.0.0`

## Resumo
Primeira release estável do backend com autenticação JWT, RBAC, CRUDs de domínio (usuários, autores, categorias, livros), fluxo de empréstimo/devolução, seed inicial, testes automatizados e infraestrutura Docker.

## Novidades

### API e regras de negócio
- Cadastro e login:
  - `POST /auth/register`
  - `POST /auth/login`
- Autorização por perfil (`ROLE_ADMIN`) com middleware.
- CRUD de autores, categorias e livros.
- Cadastro de leitores por admin (`POST /users`) com senha gerada automaticamente.
- Empréstimos e devoluções:
  - `POST /loans`
  - `POST /loans/:id/return`
- Listagens:
  - ativos (`GET /loans/active`)
  - atrasados (`GET /loans/overdue`)
  - histórico (`GET /loans/history`)
  - área do usuário (`GET /loans/me`)
- Regras implementadas:
  - senha forte no cadastro
  - e-mail e título únicos
  - livro com categoria obrigatória e ao menos 1 autor
  - bloqueio de exclusão para vínculos ativos
  - cálculo de atraso na devolução

### Persistência
- Modelagem Prisma completa:
  - `User`, `Category`, `Author`, `Book`, `BookAuthor`, `Loan`
- Enums de status:
  - `UserStatus`, `BookStatus`, `LoanStatus`
- Seed idempotente:
  - usuário admin
  - categorias e autores de exemplo
  - livro de exemplo (`1984`)

### Qualidade e arquitetura
- Tratamento centralizado de erros com `AppError`.
- Cobertura de testes ampliada:
  - auth
  - middlewares
  - utils
  - books
  - loans

### Infraestrutura
- `Dockerfile` para API.
- `docker-compose.yml` com API + PostgreSQL.

## Como atualizar / subir esta versão
```bash
npm install
npm run prisma:generate
npm run prisma:migrate
npm run prisma:seed
npm run build
npm test
npm run dev
```

Com Docker:
```bash
docker-compose up --build
```

## Credenciais padrão do seed (dev)
- E-mail: `admin@biblioteca.local`
- Senha: `Admin@123`

## Limitações conhecidas
- Envio de senha por e-mail no cadastro de leitor não implementado (senha retornada na resposta da API).
- Paginação/ordenação avançada nas listagens não implementada nesta release.
